Constant CONTAINER_ROLES

pub const CONTAINER_ROLES: [Role; 35];
Expand description

List of all “container roles”. This is not to be confused with any role that can contain children, or roles without semantic meaning. This constant is for heuristics reasons, in order to determine whether we should speak the entire contents of a subtree or not. Subtrees are rather expensive (latency-wise) to compute constantly, and in some cases, we will get a [atspi::StateChangedEvent] which tells us that a document is focused (getting an entire document subtree can potentially take multiple seconds).

If the role of the item is contained within this list, it is a hint to not try to grab the entire subtree, but rather just to say the name of the container (or sometimes the description).
